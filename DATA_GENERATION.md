# Data Generation Guide

This file explains how data moves through this project, how to generate the files the frontend needs, and which commands to run for common cases such as `Garmin-CN`.

## Overview

The project is an offline-generated static site.

The data flow is:

```text
Sports platform / local GPX-TCX-FIT
  -> Python sync script
  -> SQLite database: run_page/data.db
  -> static JSON: src/static/activities.json
  -> optional SVG assets: assets/*.svg
  -> frontend build / GitHub Pages
```

The frontend does not fetch data from Garmin, Strava, or other platforms at runtime.
It reads generated files from the repository.

## Key Files

- `run_page/data.db`
  - Main SQLite database.
- `src/static/activities.json`
  - Main JSON consumed by the React frontend.
- `assets/*.svg`
  - Pre-rendered posters and yearly charts.
- `GPX_OUT/`, `TCX_OUT/`, `FIT_OUT/`
  - Downloaded or imported activity files.

The output paths are defined in [run_page/config.py](/Users/shouyong/myqz/code/2026/running_page/run_page/config.py:1).

## What Actually Generates `activities.json`

The frontend data file is generated by `make_activities_file(...)` in [run_page/utils.py](/Users/shouyong/myqz/code/2026/running_page/run_page/utils.py:51).

That function:

1. Opens `run_page/data.db`
2. Imports activities from GPX/TCX/FIT or app data
3. Normalizes them through `Generator`
4. Writes `src/static/activities.json`

If `src/static/activities.json` is missing, the most direct rebuild command is usually:

```bash
python3 run_page/gpx_sync.py
```

This imports all files from `GPX_OUT/` and regenerates `src/static/activities.json`.

## Garmin-CN Workflow

### 1. Generate Garmin secret string

```bash
python3 run_page/get_garmin_secret.py <email> <password> --is-cn
```

This prints a `secret_string`.
Save it. This is what you pass to `garmin_sync.py` or store in GitHub Secrets.

### 2. Sync Garmin-CN data locally

All activities:

```bash
python3 run_page/garmin_sync.py "<secret_string>" --is-cn
```

Running only:

```bash
python3 run_page/garmin_sync.py "<secret_string>" --is-cn --only-run
```

What this does:

- Downloads activities from Garmin China
- Stores files in `GPX_OUT/` by default
- Updates `run_page/data.db`
- Writes `src/static/activities.json`

### 3. Start frontend locally

```bash
pnpm dev
```

The frontend imports `src/static/activities.json` directly in [src/hooks/useActivities.ts](/Users/shouyong/myqz/code/2026/running_page/src/hooks/useActivities.ts:1).

## Rebuild JSON from Existing GPX Files

If you already have GPX files in `GPX_OUT/`, you do not need to call Garmin again.

Just run:

```bash
python3 run_page/gpx_sync.py
```

This is the shortest repair path when:

- sync downloaded files successfully
- `run_page/data.db` exists
- but `src/static/activities.json` is missing

## Common Problem: Reverse Geocoding Timeout

During import, the project may call `nominatim.openstreetmap.org` to fill `location_country`.
If your network is slow or blocked, you may see repeated timeout logs like:

```text
Retrying after connection broken by 'ReadTimeoutError(...)'
```

Two environment variables are now supported in this repo to make this easier:

- `SKIP_REVERSE_GEOCODE=true`
  - Skip reverse geocoding entirely.
- `GEOPY_TIMEOUT=10`
  - Increase geocoding timeout in seconds.

Recommended recovery command:

```bash
SKIP_REVERSE_GEOCODE=true python3 run_page/gpx_sync.py
```

If you want to keep reverse geocoding but increase timeout:

```bash
GEOPY_TIMEOUT=10 python3 run_page/gpx_sync.py
```

If you skip reverse geocoding:

- JSON generation succeeds more reliably
- location fields may be empty or less complete
- core activity stats and charts still work

## Generate SVG Assets

### Generate aggregate GitHub-style chart

```bash
python3 run_page/gen_svg.py --from-db --type github --output assets/github.svg
```

### Generate yearly GitHub-style charts

```bash
python3 run_page/gen_svg.py --from-db --type github --output assets/github.svg --generate-all-years
```

This generates files like:

- `assets/github_2021.svg`
- `assets/github_2022.svg`
- `assets/github_2023.svg`

### Generate a single year only

```bash
python3 run_page/gen_svg.py --from-db --type github --year 2026 --output assets/github_2026.svg
```

### Other common SVG outputs

```bash
python3 run_page/gen_svg.py --from-db --type grid --output assets/grid.svg
python3 run_page/gen_svg.py --from-db --type circular --output assets/year.svg
python3 run_page/gen_svg.py --from-db --type monthoflife --output assets/mol.svg
python3 run_page/gen_svg.py --from-db --type year_summary --output assets/year_summary.svg
```

## Minimal Local Workflow

If you want the shortest end-to-end local workflow:

```bash
pip3 install -r requirements.txt
corepack enable
pnpm install

python3 run_page/get_garmin_secret.py <email> <password> --is-cn
python3 run_page/garmin_sync.py "<secret_string>" --is-cn --only-run

python3 run_page/gen_svg.py --from-db --type github --output assets/github.svg --generate-all-years
pnpm dev
```

If geocoding gets stuck:

```bash
SKIP_REVERSE_GEOCODE=true python3 run_page/gpx_sync.py
```

## GitHub Actions Flow

The automation entrypoint is [run_data_sync.yml](/Users/shouyong/myqz/code/2026/running_page/.github/workflows/run_data_sync.yml:1).

For Garmin-CN, the relevant settings are:

- `RUN_TYPE: garmin_cn`
- GitHub Secret: `GARMIN_SECRET_STRING_CN`

The workflow will run:

```bash
python run_page/garmin_sync.py ${{ secrets.GARMIN_SECRET_STRING_CN }} --is-cn
```

If you want running only, change that step to:

```bash
python run_page/garmin_sync.py ${{ secrets.GARMIN_SECRET_STRING_CN }} --is-cn --only-run
```

For GitHub Pages, data sync and site publish are separate steps:

1. `Run Data Sync` updates `data.db`, `activities.json`, and assets
2. `Publish GitHub Pages` builds and deploys the static site

## When to Use Which Script

- Use `garmin_sync.py`
  - when Garmin-CN or Garmin is your source of truth
- Use `gpx_sync.py`
  - when files already exist in `GPX_OUT/`
- Use `gen_svg.py`
  - when JSON exists and you want refreshed charts
- Use `fix_location.py`
  - when the site works, but location text is incomplete and you want to repair it later

## Practical Advice

- Treat `run_page/data.db` as the canonical source inside this repo.
- Treat `src/static/activities.json` as the frontend cache.
- Treat `assets/*.svg` as pre-rendered view assets.
- If JSON is missing, fix JSON generation first.
- If charts are missing, run `gen_svg.py` after the database is ready.
- If geocoding is unstable, skip it first and repair location later.
